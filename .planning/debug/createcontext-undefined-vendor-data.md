## Resolution

root_cause: "Rollup 4 deduplicates inline TypeScript runtime helpers (__assign, __rest, __spreadArray, __importDefault — generated by tsc from all CJS TypeScript packages like @tanstack, @supabase, @radix-ui, recharts, react-leaflet) across manual chunks. When vendor-react is isolated from these packages, Rollup extracts the shared interop helper into whichever chunk uses it most, then vendor-react must import it from there. This always creates a circular dependency (vendor-react -> vendor-X -> vendor-react). Because packages like @tanstack/react-query and react-leaflet call React.createContext() at module evaluation time (top-level code), the circular import causes React to be undefined when those lines execute -> TypeError."
fix: "Consolidated all node_modules into a single vendor-libs chunk in vite.config.ts. When all CJS packages share the same chunk, the interop helper stays local to that chunk — no cross-chunk import is needed, no circular dependency is possible. vendor-libs (1.65MB unminified, 497KB gzip) is self-contained and has zero cross-vendor imports."
verification: "Build produces: vendor-libs-*.js (no cross-vendor imports), index-*.js (imports only from vendor-libs). Zero circular dependencies detected via static analysis. TypeScript check passes. 57 lazy chunks all import only from vendor-libs."
files_changed: "vite.config.ts"
